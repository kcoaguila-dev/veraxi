# 🎯 Landing Page Generation Skill

When asked to design, build, or write copy for a landing page in the Veraxi repository, you **MUST** follow the rules below. This prevents generating generic "AI slop" and ensures all marketing assets use professional, high-converting frameworks.

## 1. Copywriting Framework (The Brains)
You must synthesize the Julian Shapiro and StoryBrand (SB7) frameworks for all copy.

### The Core Philosophy:
*   **Customer = Hero:** Never talk about the product first. Always center the user's problem (e.g., Data Silos, AI hallucinations, Context limits).
*   **Clarity > Cleverness:** Never use corporate fluff (e.g., "synergy," "next-gen," "revolutionize"). Write at a 5th-grade reading level.
*   **The Value Proposition:** Use the Julian Shapiro structure:
    1.  **Bad Alternative:** Highlight why the status quo sucks.
    2.  **Better Solution:** Explain exactly how Veraxi solves it.
    3.  **Action Statement:** Give a clear, benefit-driven CTA.

## 2. Structural Requirements (The Flow)
Every landing page must follow this exact vertical flow:

1.  **Hero Section (Above the Fold):**
    *   Headline: Clear, outcome-focused (e.g., "Give your AI agents a brain.").
    *   Sub-headline: Contextualizes the headline (e.g., "Turn messy data silos into a searchable Knowledge Layer").
    *   Primary CTA: Action-oriented (e.g., "Start Building Free").
    *   Visual: A concrete "Show, Don't Tell" element (e.g., Architecture diagram, Code Snippet, Terminal window). No abstract brain illustrations.
2.  **Trust & Authority (Social Proof):**
    *   Ecosystem Logos: Display integrations (Claude, Cursor, Neo4j, Qdrant, Flutter, OpenAI).
3.  **The Problem:**
    *   Agitate the "villain" (e.g., why local LLMs fail without context).
4.  **The Guide & Plan:**
    *   Present the solution in 3 dead-simple steps (e.g., 1. Connect DB, 2. Run MCP, 3. Chat).
5.  **Final CTA:**
    *   Remind them of the success they will achieve.

## 3. Aesthetic Rules (The Beauty)
All UI code (Flutter, CSS, or SVG mockups) must adhere to the global Veraxi guidelines:

*   **Theme:** Premium Dark Mode (Backgrounds in `#171717` or `#212121`).
*   **Typography:** Google Font `Inter`.
*   **Color Palette:** Use `#3B82F6` (Blue) or `#8B5CF6` (Purple) for primary accents and gradients. Do not use plain primary colors.
*   **Effects:** Use glassmorphism for cards and subtle glow effects behind key graphics.
*   **Interactions:** If building in Flutter/Web, include micro-animations for hover states and transitions.

## 4. Execution Protocol
1.  Read the user's specific request.
2.  Draft the copy using the frameworks above before writing code.
3.  If building UI, ensure the code strictly matches the Aesthetic Rules.
